# pages-cns-mk

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容相互独立，不依附于任何特定域名或网站。仓库的主要目的是提供一个结构化的存储空间，便于历史页面的集中管理与访问。

## 项目简介

`pages-cns-mk` 是一个以静态 HTML 页面为核心的归档仓库。仓库中的每个页面均为独立的 HTML 文件，可单独打开或嵌入其他环境。项目不涉及动态后端或数据库，仅依赖 GitHub Pages 进行页面托管。

## 目录说明

仓库目录结构如下（示例，实际以仓库内容为准）：

```
pages-cns-mk/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 归档页面存放目录
│   ├── page-example-1.html
│   ├── page-example-2.html
│   └── ...
├── assets/             # 公共资源（如 CSS、图片、字体等）
│   ├── styles.css
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立归档的 HTML 页面。建议按时间、主题或类型组织子目录。
- `assets/`：存放页面共用的静态资源，以减小重复引用。
- 根目录下的 `index.html` 可作为导航或说明页面，非必需。

## 页面归档说明

- 每个页面均以完整的 HTML 文档形式存在，包含 `<html>`、`<head>`、`<body>` 等基本结构。
- 页面之间无强依赖关系，可独立运行。
- 归档内容不涉及任何具体网站或服务的模拟、替代或推广。
- 页面可能包含基本的样式与脚本，但以轻量、自包含为原则。

### 添加新页面

1. 将新 HTML 文件放入 `pages/` 目录（或相应子目录）。
2. 确保文件路径与引用资源（如 CSS、图片）的相对路径正确。
3. 提交 Pull Request 或直接推送至 main 分支（视仓库权限而定）。
